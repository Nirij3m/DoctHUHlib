Du client jusqu'au serveur, ça donne ça :
- Le client demande une page (i.e: /login).
- Le serveur utilise le ROUTEUR (index.php, à la racine du dossier) pour savoir où rediriger.
- Le CONTROLEUR effectue tout le côté "code" du php, et est fini par un require de la vue.
- La vue contient le contenu HTML SEULEMENT. On peut mettre du PHP si on a besoin d'AFFICHER quelque chose (i.e: la pfp de l'utilisateur, récupérer une variable, ...)

Et pour la structure :
assets: On mettra le css, js, les images dedans, chacun dans un dossier.
src: contient les dao (database access), les view, et les applis (controleurs et utils).

Ça donnera donc :
assets
    js
    css
    images
src
    appli
        cntrlApp.php
        cntrlLogin.php
    dao
        daoUser.php
        daoPlanning.php
        ...
    view
        alerts.php
        header.php
        footer.php (?)
        v<page>.php
index.php


/!\ Utils contient toutes les constantes que tu voudras modifier pour accéder à la DB ou autre.
/!\ Une DAO par TABLE. (copie/colle un fichier dao et modifies le nom et les fonctions, ça suffit).
